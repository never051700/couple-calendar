# 보안 안내

## 취약점 제보

인증 우회, 다른 사용자의 일정 접근, 푸시 토큰 노출, 계정 삭제 오류처럼 사용자
데이터에 영향을 줄 수 있는 문제는 공개 Issue로 작성하지 마세요. 저장소 소유자와
기존에 합의한 메신저 등 비공개 채널로 다음 내용을 전달해 주세요.

- 영향을 받는 앱 버전과 플랫폼
- 재현 순서
- 실제 개인정보를 제거한 증거
- 예상 영향

수정과 자격 증명 교체가 끝난 뒤 공개 가능한 범위만 Issue나 변경 기록으로
남깁니다.

## 비밀정보 노출 시 대응

1. 노출된 키나 토큰을 즉시 폐기합니다.
2. Supabase, Expo/EAS, Firebase 또는 Apple에서 새 자격 증명을 발급합니다.
3. EAS와 Supabase 환경 값을 교체합니다.
4. Git 기록과 Actions 로그에서 노출 범위를 확인합니다.
5. 양쪽 앱의 로그인·DB·푸시 기능을 다시 검사합니다.

파일을 최신 커밋에서 삭제하는 것만으로는 이미 공개된 비밀정보를 보호할 수
없습니다.
